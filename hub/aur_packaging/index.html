AUR Packaging Guide


  • Create a directory for your packages if it does not exist. Call it AUR.
    • mkdir $HOME/AUR
  • Generate an SSH key pair named aur: 
    ssh-keygen -f ~/.ssh/aur
  • Copy the public key and add it to your AUR account: https://aur.archlinux.org
  • Configure SSH for authentication: 
    nano ~/.ssh/config
    Host aur.archlinux.org
     IdentityFile ~/.ssh/aur
     User aur
  • Clone the repository for your package: 
    cd ~/AUR
git clone ssh://aur@aur.archlinux.org/pkgname.git
cd pkgname/
    Note:
    - If it’s a new package, Git will say "You have cloned an empty directory".
    - If it’s an existing package, you’ll have the repo with all files already there.
  • Go into the cloned repository and add a .gitignore file with * inside. This ignores everything unless you force-add it with git add -f filename.
  • Sanity-check the PKGBUILD using namecap: 
    namecap PKGBUILD
    No results mean your PKGBUILD is good to go.
  • Build the package to ensure it compiles correctly. Update the SHA256 sums: 
    updpkgsums
  • Every time before pushing, update .SRCINFO: 
    makepkg --printsrcinfo > .SRCINFO
  • Add the PKGBUILD and .SRCINFO files: 
    git add -f PKGBUILD .SRCINFO
  • Commit your changes: 
    git commit -m "Your commit message"
  • Push to the AUR: 
    git push

Et voilà
